登陆, 注册

  1. 打开http://sdkui.egamingfund.org?merchantId=xxx&callbackUrl=xxx

    • xxx指的是参数值
    • merchantId是账号id
    • callbackUrl是登陆成功后跳转的地址

  2. 用户登陆注册成功之后, 点击返回商户页面跳转到callbackUrl的地址并且登陆参数会拼接在连接后面

    • nickname玩家昵称

    • timestamp登陆时间

    • token令牌

    • userId玩家id

    • sign签名.

      • 厂商可以通过这个签名验证用户信息的有效性
      • 签名采用32位md5小写的形式
      • sign= md5("nickname=x&timestamp=x&token=x&userId=x&secretKey=x")

支付

  1. 向eGF请求创建订单

    • 地址: http://sdk.egamingfund.org/sdk/createOrder

    • http method: POST

    • header: Content-Type = application/json;charset=utf-8

    • request params: 当前一律采用utf-8编码格式

      • merchantOrderId: string, 商户的唯一订单id, 必须

      • money: string, GFG*10^8, 必须

      • merchantId: long, 商户的用户id, 必须

      • payType: string, 付款方式, 必须是GFG, 必须

      • asyncNoticeUrl: string, 订单回调通知URL, 访问方式必须是GET, 必须

      • subject: string, 商品主题, 必选

      • body: string, 商品描述, 可选

      • extra: string, 商户备注, 会在通知的时候还给商户, 可选

      • charset: string, 编码, 必须是utf-8, 必选

      • sign: 签名, 用于验证商户操作的合法性, 必须

        • eGF可以通过这个签名验证厂商订单的的有效性
        • 签名采用32位md5小写的形式
        • sign= md5("asyncNoticeUrl=x&body=x&charset=x&extra=x&merchantId=x&merchantOrderId=x&money=x&payType=x&subject=x&secretKey=x")
        • 如果可选的参数为null, 参数名也需要参与签名. 比如. extra=null, sign= md5("asyncNoticeUrl=x&body=x&charset=x&extra=&merchantId=x&merchantOrderId=x&money=x&payType=x&subject=x&secretKey=x")

    • response params: 当前一律采用utf-8编码格式

      • code: 响应码

        • 1 = 成功
        • 2 = 失败
        • 3 = 缺失参数
        • 4 = 商家不存在
        • 5 = 验证码错误
        • 6 = 重复商家订单
        • 7 = 钱格式不正确
        • 8 = 支付方式不正确
        • 9 = 编码格式不正确

      • msg: 提示语

      • data

        • otcOrderId: string, sdk订单号
        • timestamp: long, gf端订单创建时间
        • merchantOrderId: string, 你的订单号
        • extra: string, 你的额外信息

  2. 用户浏览器跳转sdk支付页面

    • 地址: http://sdkui.egamingfund.org?orderId=xxx&callbackUrl=xxx&money=xxx&merchantName=xxx

      • xxx指的是参数值
      • orderId: sdk的订单id
      • callbackUrl: 支付成功或者失败后跳转的地址
      • money: 界面显示金额, 只是起到显示作用, 用户实际扣款只与创建订单时的money有关
      • merchantName: 界面显示收款人姓名, 只是起到显示作用

  3. 用户支付成功后, 厂商在createOrder时asyncNoticeUrl接口会接收到eGF的支付成功的通知

    • eGF调用方式:

      • method: GET

      • url: asyncNoticeUrl?extra=x&merchantId=x&merchantOrderId=x&money=x&otcOrderId=x&payeeId=x&payerId=x&payType=x&sign=x

        • extra: 创建订单时携带的备注信息, 返回URLEncode(extra)之后的extra

        • merchantId: 厂商id

        • merchantOrderId: 厂商的订单编号

        • money: 订单金额, GFG*10^8

        • otcOrderId: eGF订单id

        • payType: 支付方式, 当前只能是GFG

        • payeeId: 收款人的eGFid

        • payerId: 付款人的eGFid

        • sign: 签名,

          • 只有签名验证通过, 才能确认是eGF的通知
          • 签名采用32位md5小写的形式
          • sign= md5("extra=x&merchantId=x&merchantOrderId=x&money=x&otcOrderId=x&payeeId=x&payerId=x&payType=x&secretKey=x")
          • 如果可选的参数为null, 参数名也需要参与签名. 比如. extra=null, sign= md5("extra=&merchantId=x&merchantOrderId=x&money=x&otcOrderId=x&payeeId=x&payerId=x&payType=x&secretKey=x")

    • 通知频率

      • 以第一次进入线程为起始时间，一分钟前，每十秒通知三次
      • 一分钟至五分钟，每分钟通知三次
      • 五分钟至半小时，每五分钟通知三次
      • 半小时过至一天，每半小时通知三次
      • 一天过后不通知

厂商转账

这里指厂商将自己账号中的GFG转到目标用户的账号中

  1. 向eGF请求向目标用户转账

    • 地址: http://sdk.egamingfund.org/sdk/merchantPayToUser

    • http method: POST

    • header: Content-Type = application/json;charset=utf-8

    • request params: 当前一律采用utf-8编码格式

      • merchantOrderId: string, 商户的唯一订单id, 必须

      • merchantId: long, 商户的用户eGFID, 必须

      • withdrawUserId: long, 目标用户的eGFID, 必须

      • money: string, GFG*10^8, 必须

      • payType: string, 付款方式, 必须是GFG, 必须

      • asyncNoticeUrl: string, 订单回调通知URL, 访问方式必须是GET, 必须

      • extra: string, 商户备注, 会在通知的时候还给商户, 可选

      • charset: string, 编码, 必须是utf-8, 必选

      • sign: 签名, 用于验证商户操作的合法性, 必须

        • eGF可以通过这个签名验证厂商订单的的有效性
        • 签名采用32位md5小写的形式
        • sign= md5("asyncNoticeUrl=x&charset=x&extra=x&merchantId=x&merchantOrderId=x&money=x&payType=x&withdrawUserId=x&secretKey=x")
        • 如果可选的参数为null, 参数名也需要参与签名. 比如. extra=null, sign= md5("asyncNoticeUrl=x&charset=x&extra=&merchantId=x&merchantOrderId=x&money=x&payType=x&withdrawUserId=x&secretKey=x")

    • response params: 当前一律采用utf-8编码格式

      • code: 响应码

        • 41 = 成功
        • 42 = 失败
        • 43 = 缺失参数
        • 44 = 商家不存在
        • 45 = 提现用户不存在
        • 46 = 验证码错误
        • 47 = 重复商家订单
        • 48 = 钱格式不正确
        • 49 = 余额不足
        • 50 = 支付方式不正确
        • 51 = 编码格式不正确

      • msg: 提示语

      • data

        • otcOrderId: string, sdk订单号
        • timestamp: long, gf端订单创建时间
        • merchantOrderId: string, 你的订单号
        • extra: string, 你的额外信息

  2. 厂商转账成功后, 厂商在asyncNoticeUrl接口会接收到eGF的转账成功的通知

    • eGF调用方式:

      • method: GET

      • url: asyncNoticeUrl?extra=x&merchantId=x&merchantOrderId=x&money=x&otcOrderId=x&payeeId=x&payerId=x&payType=x&sign=x

        • extra: 创建订单时携带的备注信息, 原封不动返回, 返回URLEncode(extra)之后的extra

        • merchantId: 厂商id

        • merchantOrderId: 厂商的订单编号

        • money: 订单金额, GFG*10^8

        • otcOrderId: eGF订单id

        • payType: 支付方式, 当前只能是GFG

        • payeeId: 收款人的eGFid

        • payerId: 付款人的eGFid

        • sign: 签名,

          • 只有签名验证通过, 才能确认是eGF的通知
          • 签名采用32位md5小写的形式
          • sign= md5("extra=x&merchantId=x&merchantOrderId=x&money=x&otcOrderId=x&payeeId=x&payerId=x&payType=x&secretKey=x")
          • 如果可选的参数为null, 参数名也需要参与签名. 比如. extra=null, sign= md5("extra=&merchantId=x&merchantOrderId=x&money=x&otcOrderId=x&payeeId=x&payerId=x&payType=x&secretKey=x")

    • 通知频率

      • 以第一次进入线程为起始时间，一分钟前，每十秒通知三次
      • 一分钟至五分钟，每分钟通知三次
      • 五分钟至半小时，每五分钟通知三次
      • 半小时过至一天，每半小时通知三次
      • 一天过后不通知

厂商订单查询

  1. 厂商可以通过提交订单时的merchantOrderId来查询

    • 地址: http://sdk.egamingfund.org/sdk/queryMerchantOrderStatus

      • http method: POST

      • header: Content-Type = application/json;charset=utf-8

      • request params: 当前一律采用utf-8编码格式

        • merchantId: long, 厂商id, 必须

        • merchantOrderId: string, 厂商订单id, 必须

        • sign: string, 用于验证商户操作的合法性, 必须

          • eGF可以通过这个签名验证厂商订单的的有效性
          • 签名采用32位md5小写的形式
          • sign= md5("merchantId=x&merchantOrderId=x&secretKey=x")

      • response params: 当前一律采用utf-8编码格式

        • code: 响应码

          • 成功 = 170
          • 失败 = 171
          • 缺失参数 = 172
          • 不存在该订单 = 173
          • 不存在该厂商 = 174
          • 签名错误 = 175

        • msg: 提示语

        • data

          • merchantId: 厂商id , long

          • merchantOrderId: 厂商订单id , string

          • otcOrderId: eGF订单id , string

          • money: gfg*10^8, 正数表示厂商收钱， 负数表示商户给钱, 必须 , string

          • payeeId: 收款人的eGFid , long

          • payerId: 付款人的eGFid , long

          • payType: 付款方式(支付渠道), gf, 必须 , string

          • extra: 商户备注, 会在通知的时候还给商户 , string

          • status: 单状态, byte

            • 0 = 待支付
            • 1 = 已支付
            • 5 = 未知错误
